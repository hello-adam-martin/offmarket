# Phase 3 — UI Design Contract: Public Pages

> Visual and interaction contract for Phase 3. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> This phase is a pure token migration and AI slop removal pass over seven public-facing pages. All design infrastructure (tokens, fonts, dark mode, component classes) is established from Phases 1 and 2. No new libraries, no new component classes, and no backend changes are required.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none — no shadcn, no component library |
| Preset | not applicable |
| Component library | Headless UI 2.2.9 (existing, used for FAQ accordion toggle via useState) |
| Icon library | none specified — use inline SVG or existing icon patterns from pages |

## Spacing Scale

Declared values (base unit: 4px — source: DESIGN.md):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, tight inline padding |
| sm | 8px | Compact element spacing (step number gutter, badge padding) |
| md | 16px | Default element spacing, form field padding, card internal padding |
| lg | 24px | Section internal padding, card generous padding |
| xl | 32px | Between-component spacing, layout gaps |
| 2xl | 48px | Between-section spacing on homepage |
| 3xl | 64px | Page-level vertical rhythm (`py-16` = 64px) |

Exceptions:
- Touch targets: minimum 44px enforced via `min-h-[44px]` on all interactive buttons (WCAG 2.5.5). Not a spacing token.
- Hairline separators: use CSS `border` (1px default) — not a spacing token.
- Legal page section separators: `mb-8` (32px) between h2 sections, `mb-4` (16px) between paragraphs.

**Source:** Phase 1 UI-SPEC.md (Spacing section); DESIGN.md Spacing section.

## Typography

Phase 3 declares 4 active sizes. Sizes 12px, 14px, and 18px are part of the system-wide scale established in Phase 1 but are not primary sizes for Phase 3 pages.

| Role | Size | Font Family | Weight | Line Height | Usage in Phase 3 |
|------|------|-------------|--------|-------------|------------------|
| Display | 48px (3xl) | General Sans (`font-display`) | 700 | 1.1 | Homepage hero headline only |
| Heading | 32px (2xl) | General Sans (`font-display`) | 700 | 1.2 | Homepage section headings (h2), legal page h1, help page h1 |
| Subheading | 24px (xl) | General Sans (`font-display`) | 700 | 1.3 | Homepage subsections, legal page h2, auth card heading |
| Body | 16px (md) | DM Sans (`font-sans`) | 400 | 1.6 | Body copy, FAQ answers, legal body text, auth form, metadata, captions |

**System reference (inherited from Phase 1, not primary in Phase 3):** 18px lead paragraphs, 14px secondary/caption text, 12px badge labels. Apply these sizes directly via Tailwind scale values (`text-sm`, `text-xs`) where needed — they do not require declaration as primary type roles in this phase.

**Weights declared (2 weights across all families):**
- 700 bold — all headings (Display, Heading, Subheading) in General Sans
- 400 regular — all body copy, metadata, and UI text in DM Sans

**Data rendering rule:** Any numeric value shown in the homepage hero stats row, stats section, or DemandChecker result must apply `tabular-nums` (Tailwind `tabular-nums` utility class). Source: FOUN-14 requirement.

**Font class pattern:**
- Headings: always include `font-display` to apply General Sans
- Body: `font-sans` is default (applied at root via `layout.tsx`) — no explicit class needed on body text

**Source:** DESIGN.md Typography section; Phase 1 UI-SPEC.md Typography section; RESEARCH.md Code Examples.

**Accent reserved for (explicit list):**
- `.btn-primary` CTA button fill (`bg-accent`)
- Active FAQ category tab fill (`bg-accent text-white`)
- Open FAQ chevron icon (`text-accent`)
- Links in legal pages and auth disclaimer (`text-accent hover:text-accent-hover`)
- Quick action card icons in help page (`text-accent`)
- How It Works step numbers (`text-accent`)
- DemandChecker check button (`bg-accent` via `.btn-primary`)
- Active filter/tag indicators (inherited from Phase 2 BrowsePageClient — no changes needed)
- Homepage CTA buttons below hero stats (`btn-primary`)

**Accent NOT used for:**
- Hero section background — always `bg-bg`
- Section panel backgrounds — always `bg-bg` or `bg-surface-raised` (never `bg-accent-50`, `bg-primary-50`)
- General decorative elements

## Page-Level Contracts

### Homepage (`apps/web/src/app/page.tsx`)

**Hero section (PUBL-01, D-01 through D-05):**
- Background: `bg-bg py-16` — no gradient, no SVG pattern, no colored background
- Layout: `max-w-content mx-auto px-4 sm:px-6 lg:px-8`
- Heading: left-aligned, `text-3xl font-display font-bold text-text-base` — replace "Find Your Dream Home" with data-forward copy
- Stats: server-side fetched aggregate demand data (active buyers, active regions, avg budget) rendered with `tabular-nums`. If fetch fails: D-04 fallback — static left-aligned value proposition copy, no numbers, no skeleton
- CTAs: two `.btn-primary` buttons below stats, left-aligned
- Remove entirely: gradient overlay, SVG cross-hatch pattern, trust indicators row (checkmarks)

**Section headings (D-06):** All `<h2>` across every section: `text-2xl font-display font-bold text-text-base mb-4` — no `text-center`, no `mx-auto` centering on text

**How It Works (D-07):**
- Remove icon-in-colored-circle elements entirely
- Numbered steps using `<ol>` with accent step numbers: `text-xl font-display font-bold text-accent w-6 flex-shrink-0`
- Section background: `bg-bg` only — no `bg-primary-50` or `bg-accent-50` panels

**Stats row (D-08):** Keep grid layout. Apply `tabular-nums` to all numeric values. Replace all `gray-*` tokens. Use `max-w-content`.

**Demand Checker section (D-09):**
- Remove "For Property Owners" pill badge
- Left-align section heading
- Container uses `max-w-content` and `bg-bg`
- Inner `DemandChecker` component restyled per component contract below

**Pricing section:** Keep section. Pro tier card: `bg-accent text-white rounded-lg` replacing `bg-primary-600` gradient card. Free and Owner tiers: `.card border-2 border-border rounded-lg`. All `rounded-2xl` → `rounded-lg`.

**Testimonials section:** Remove entirely. AI slop blacklist violations (fake quotes, star ratings, avatar initials). No replacement section needed — tighten spacing between surrounding sections with `py-16` on adjacent sections.

**Benefits grid section:** Remove icon-in-colored-circle pattern. If section retained, replace with plain text list using `text-text-base` headings and `text-text-secondary` descriptions inside `.card` wrappers.

**Final CTA section:** Remove gradient background. Use `bg-bg` or `bg-primary` (dark navy). Replace "Ready to Find Your Dream Home?" copy with DESIGN.md compliant messaging. Apply `.btn-primary` to CTA button.

**Container rule (D-10):** Every `max-w-7xl` → `max-w-content`. No exceptions on this page.

## Layout Contracts

**Max content width:** `max-w-content` (1120px) on every page section wrapper. Never `max-w-7xl`, `max-w-4xl`, or `max-w-3xl`. Source: D-10, RESEARCH.md Pitfall 1.

**Container pattern:**
```
max-w-content mx-auto px-4 sm:px-6 lg:px-8
```

**Section vertical spacing (homepage):**
- Hero: `py-16`
- Interior sections: `py-16`
- Between sections: no explicit margin — rely on `py-16` pairs creating 32px visual gap

**Responsive breakpoints:** sm(640px), md(768px), lg(1024px), xl(1280px). All existing responsive classes preserved. Changes are token-only — no breakpoint restructuring unless a specific breakpoint produces a slop pattern.

**Text alignment rule (D-06):** All section headings and body content left-aligned. `text-center` is forbidden except:
- Auth card heading (centered within a centered card — UX exception D-13)

**Dark mode:** All semantic tokens auto-adapt via CSS variable overrides in `.dark` class (Phase 1 globals.css). No per-component `dark:` overrides needed when using correct semantic tokens. Verify every replaced token uses a semantic token (never a hardcoded color).
