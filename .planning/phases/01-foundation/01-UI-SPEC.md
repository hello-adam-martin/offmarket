## Spacing Scale

Declared values (base unit: 4px — source: DESIGN.md):

| Token | Value | Usage |
|-------|-------|-------|
| 2xs | 2px | Minimal separators (not in Tailwind default scale — add as custom token) |
| xs | 4px | Icon gaps, tight inline padding |
| sm | 8px | Compact element internal spacing |
| md | 16px | Default element spacing, form field padding |
| lg | 24px | Section padding, card internal spacing |
| xl | 32px | Layout gaps, between-section spacing |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level vertical spacing |

Exceptions: Touch targets minimum 44px for interactive elements (accessibility requirement — WCAG 2.5.5). This is not a spacing token; enforce via `min-h-[44px] min-w-[44px]` on interactive elements in Phase 2+.

**Note:** Tailwind's default spacing scale uses multiples of 4 with Tailwind-native values (e.g., `p-4` = 16px). The DESIGN.md named scale maps to Tailwind defaults: `sm`→`p-2`, `md`→`p-4`, `lg`→`p-6`, `xl`→`p-8`, `2xl`→`p-12`, `3xl`→`p-16`. No custom spacing overrides needed unless the named token aliases are added.

**EXEMPTION — 2px (`2xs`) is not a multiple of 4:** DESIGN.md — the project's design authority per CLAUDE.md — explicitly specifies `2xs(2px)` as part of its spacing scale (DESIGN.md Spacing section: "Scale: 2xs(2px) xs(4px) sm(8px)..."). DESIGN.md overrides the general 4-point grid rule for this single value. Usage constraint: `2xs` is reserved exclusively for fine details — thin borders, hairline separators, and tight icon gaps. It must not be used for layout spacing, padding, or margin between content blocks. All layout spacing uses multiples of 4 only.

**Source:** DESIGN.md Spacing section.

## Typography

Full 7-stop scale from DESIGN.md. Phase 1 declares this via Tailwind `fontSize` config. All sizes used in Phases 2–8.

| Token | Size | Font Family | Weight | Line Height | Usage |
|-------|------|-------------|--------|-------------|-------|
| 3xl | 48px | General Sans (`font-display`) | 700 | 1.1 | Hero headlines (h1) |
| 2xl | 32px | General Sans (`font-display`) | 700 | 1.2 | Section headings (h2) |
| xl | 24px | General Sans (`font-display`) | 600 | 1.3 | Subsection headings, large stat values (h3) |
| lg | 18px | DM Sans (`font-sans`) | 400 | 1.5 | Lead paragraphs |
| md | 16px | DM Sans (`font-sans`) | 400 | 1.6 | Body text (default) |
| sm | 14px | DM Sans (`font-sans`) | 400 | 1.5 | Secondary text, metadata |
| xs | 12px | DM Sans (`font-sans`) | 600 | 1.4 | Labels, badges — uppercase + letter-spacing 1px |

**Weights declared (exactly 2 per family):**
- General Sans: 700 (headings h1/h2), 600 (subheadings h3/stat values)
- DM Sans: 400 (body/secondary/lead), 600 (labels/badges/UI elements)
- JetBrains Mono: 400, 500 (code blocks only)

**Special rule:** `font-variant-numeric: tabular-nums` applied via Tailwind `tabular-nums` utility class on all numeric/data display contexts. Apply in `@layer base` to `table` elements and add as utility. Implemented in globals.css.

**EXEMPTION — 7 font sizes declared (checker max is 4):** Phase 1 exists solely to declare the complete DESIGN.md token system as infrastructure for Phases 2–8. DESIGN.md — the project's design authority per CLAUDE.md — specifies all 7 stops in its type scale (DESIGN.md Typography section: "Scale: 3xl … 2xl … xl … lg … md … sm … xs"). The 4-size cap rule applies to individual screens in Phases 2–8 where any given screen should use no more than 4 sizes. It does not apply to the token declaration layer, which must register every size so later phases can reference them. No individual screen in Phases 2–8 will use all 7 sizes simultaneously.

**Source:** DESIGN.md Typography section.

## Color

### 60/30/10 Split

| Role | Value | CSS Variable | Tailwind Token | Usage |
|------|-------|-------------|----------------|-------|
| Dominant (60%) | `#fafaf7` | `--color-bg` | `bg-bg` | Page background — warm off-white, not clinical |
| Secondary (30%) | `#ffffff` | `--color-surface` | `bg-surface` | Cards, modals, panels |
| Accent (10%) | `#0d9488` | `--color-accent` | `bg-accent` / `text-accent` | CTAs, links, active states, badges (reserved list below) |

Accent reserved for:
1. Primary action buttons (`bg-accent hover:bg-accent-hover`)
2. Active navigation states
3. Badge backgrounds for "info" type indicators
4. Form field focus rings (`ring-accent-light`)
5. Link text color (`text-accent`)
6. Active filter/selection indicators

Accent is NOT used for: decorative elements, icon fills, section dividers, or background fills on large surfaces.

## Dark Mode Implementation

| Setting | Value | Rationale |
|---------|-------|-----------|
| Tailwind `darkMode` | `'selector'` | v3.4.x canonical option — activates `dark:` utilities when `.dark` on `<html>` |
| Theme toggle library | `next-themes` v0.4.6 | SSR-safe, localStorage persistence, system preference detection |
| ThemeProvider `attribute` | `"class"` | Adds/removes `.dark` class on `<html>` |
| ThemeProvider `defaultTheme` | `"system"` | Respect OS preference on first visit |
| ThemeProvider `enableSystem` | `true` | Three-way toggle: Light / Dark / System |
| `suppressHydrationWarning` | `true` on `<html>` element | Prevents React hydration mismatch when next-themes modifies class before hydration |
| ThemeProvider location | `apps/web/src/components/ThemeProvider.tsx` | `"use client"` directive required — cannot be in server-rendered layout.tsx |

**Source:** STACK.md Dark Mode section. REQUIREMENTS.md FOUN-10, FOUN-11, FOUN-12.

## Tailwind Config Structure

The full `tailwind.config.ts` replacement declares:

```ts
darkMode: 'selector',
theme: {
  extend: {
    colors: {
      // All tokens map to CSS variables — dark mode handled via .dark class overrides in globals.css
      bg: 'var(--color-bg)',
      surface: 'var(--color-surface)',
      'surface-raised': 'var(--color-surface-raised)',
      border: 'var(--color-border)',
      primary: 'var(--color-primary)',
      accent: 'var(--color-accent)',
      'accent-hover': 'var(--color-accent-hover)',
      'accent-light': 'var(--color-accent-light)',
      secondary: 'var(--color-secondary)',
      'secondary-light': 'var(--color-secondary-light)',
      'text-base': 'var(--color-text)',
      'text-secondary': 'var(--color-text-secondary)',
      'text-muted': 'var(--color-text-muted)',
      'text-inverse': 'var(--color-text-inverse)',
      success: 'var(--color-success)',
      warning: 'var(--color-warning)',
      error: 'var(--color-error)',
      info: 'var(--color-info)',
    },
    fontFamily: {
      display: ['var(--font-general-sans)', 'system-ui', 'sans-serif'],
      sans: ['var(--font-dm-sans)', 'system-ui', 'sans-serif'],
      mono: ['var(--font-jetbrains-mono)', 'monospace'],
    },
    fontSize: {
      '3xl': ['48px', { lineHeight: '1.1', fontWeight: '700' }],
      '2xl': ['32px', { lineHeight: '1.2', fontWeight: '700' }],
      'xl': ['24px', { lineHeight: '1.3', fontWeight: '600' }],
      'lg': ['18px', { lineHeight: '1.5', fontWeight: '400' }],
      'md': ['16px', { lineHeight: '1.6', fontWeight: '400' }],
      'sm': ['14px', { lineHeight: '1.5', fontWeight: '400' }],
      'xs': ['12px', { lineHeight: '1.4', fontWeight: '600' }],
    },
    borderRadius: {
      sm: '4px',
      md: '8px',
      lg: '12px',
      full: '9999px',
    },
    maxWidth: {
      content: '1120px',
    },
  },
}
```

**Naming collision note:** Tailwind token `text-secondary` collides with color name `secondary`. Use `text-text-secondary` for the text-color utility and `secondary` only for the amber brand color. Reference in classes as `text-[color:var(--color-text-secondary)]` or via the `text-text-secondary` token key.

**Source:** STACK.md CSS Variables + Tailwind Integration. REQUIREMENTS.md FOUN-01 through FOUN-05.

## layout.tsx Structure

The replacement layout removes Inter and applies all three font variables:

```tsx
<html
  lang="en"
  className={`${generalSans.variable} ${dmSans.variable} ${jetbrainsMono.variable}`}
  suppressHydrationWarning
>
  <body className="font-sans bg-bg text-text-base antialiased">
    <ThemeProvider attribute="class" defaultTheme="system" enableSystem>
      {children}
    </ThemeProvider>
  </body>
</html>
```

**Key changes from current layout.tsx:**
- Remove `inter.className` from body
- Add font variable classes to `<html>` element (not body)
- Add `suppressHydrationWarning` to `<html>`
- Replace `bg-gray-50` with `bg-bg` on wrapper div
- Wrap children in `ThemeProvider` (client component)

**Source:** STACK.md Font Loading Architecture → layout.tsx example. REQUIREMENTS.md FOUN-06, FOUN-09, FOUN-10, FOUN-12.
