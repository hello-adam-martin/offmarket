## Spacing Scale

Inherited from Phase 1. All components in this phase use these exact values.

| Token | Value | Tailwind Equivalent | Phase 2 Usage |
|-------|-------|---------------------|---------------|
| 2xs | 2px | `gap-0.5` | Fine separators only (badge border hairlines) |
| xs | 4px | `p-1` / `gap-1` | Icon-to-text gaps, badge internal padding |
| sm | 8px | `p-2` / `gap-2` | Compact internal padding (button sm variant, inline elements) |
| md | 16px | `p-4` / `gap-4` | Default element spacing, form field padding, modal content padding |
| lg | 24px | `p-6` / `gap-6` | Card internal padding, section padding within modals |
| xl | 32px | `p-8` / `gap-8` | Layout gaps between component sections |
| 2xl | 48px | `p-12` / `gap-12` | Major breaks (footer column gaps, modal max-height breathing room) |
| 3xl | 64px | `p-16` / `gap-16` | Header-to-content vertical rhythm on larger viewports |

Exceptions:
- Touch targets: all interactive elements (buttons, toggles, nav links, close buttons) must meet `min-h-[44px]` on mobile (WCAG 2.5.5). This is a constraint, not a spacing token.
- Header height: fixed at `h-16` (64px) to match existing component; do not change.
- Mobile menu: `p-4` vertical rhythm for nav items.

**Source:** DESIGN.md Spacing section. Phase 1 UI-SPEC Spacing Scale. CONTEXT.md D-19 (button heights).

## Color

60/30/10 split inherited from Phase 1. Phase 2 component-specific accent reservation list added here.

| Role | Light Value | Dark Value | Tailwind Token | Usage |
|------|-------------|------------|----------------|-------|
| Dominant (60%) | `#fafaf7` | `#0f0f1a` | `bg-bg` | Page background behind all components |
| Secondary (30%) | `#ffffff` | `#1a1a2e` | `bg-surface` | Header background, footer background, card backgrounds, modal panels |
| Surface Raised | `#f5f5f0` | `#252540` | `bg-surface-raised` | Mobile menu panel, modal backdrop-adjacent areas, hover states on ghost items |
| Accent (10%) | `#0d9488` | `#0b8276` | `bg-accent` / `text-accent` | Reserved list below |

**Accent (`#0d9488`) is reserved for these exact elements in Phase 2:**
1. Header logo "OffMarket" text (`text-accent`)
2. Primary button background (`.btn-primary` — already in globals.css)
3. Form input focus ring (`focus:ring-accent-light focus:border-accent` — already in globals.css)
4. Dark mode toggle icon active state
5. Notification badge background (`bg-accent text-white`)
6. Active/selected region fill on NZRegionMap
7. Filter "active" indicator dot or selected filter badge background
8. Modal primary CTA button

Accent is NOT used for: header background, footer background, card backgrounds, decorative borders, icon fills on inactive states, section dividers.

**Semantic colors for badge variants:**

| Variant | Background | Text | Border | Usage |
|---------|-----------|------|--------|-------|
| info | `bg-accent-light` (`#ccfbf1` / `#134e4a` dark) | `text-accent` | none | Information, active states |
| success | `bg-success/10` | `text-success` (`#059669`) | none | Completed, active listings |
| warning | `bg-secondary-light` (`#fef3c7` / `#451a03` dark) | `text-warning` (`#d97706`) | none | Expiring, pending |
| error | `bg-error/10` | `text-error` (`#dc2626`) | none | Failed, rejected |

**Destructive color:** `#dc2626` (`text-error` / `bg-error`) — reserved exclusively for `.btn-destructive` and error validation messages. No other use.

**Border:** `border-border` — `#e5e5e0` light / `#374151` dark. Used on header separator (`border-b border-border`), card borders, input borders, modal panel borders.

**Source:** DESIGN.md Color section. Phase 1 UI-SPEC Color. CONTEXT.md D-03, D-06, D-10, D-11.

## Interaction States

Every interactive component must implement all 5 states. No state may be left unstyled.

| State | Visual Treatment |
|-------|-----------------|
| Default | As specified per component above |
| Hover | Color shift (micro 50-100ms `transition-colors`) |
| Focus | `focus:ring-2 focus:ring-accent-light focus:ring-offset-2` — teal ring |
| Active (pressed) | `active:scale-[0.98]` on buttons |
| Disabled | `opacity-50 pointer-events-none` |

Mobile touch targets: `min-h-[44px]` on all interactive elements at mobile breakpoints. Enforce via `sm:min-h-[44px]` or by using `.btn-md` / `.btn-lg` which already satisfy this.

These patterns are forbidden in all Phase 2 components:

| Forbidden | Why | Alternative |
|-----------|-----|-------------|

| Centered modal text bodies as default | Layout slop | Left-align modal body text; only headings may be centered in simple confirm dialogs |

## Dark Mode Requirements for All Components

Every component modified in Phase 2 must pass the dark mode checklist:

- No hardcoded `bg-white` — replace with `bg-surface`
- No hardcoded `bg-gray-*` — replace with `bg-surface-raised` or `bg-bg`
- No hardcoded `text-gray-*` — replace with semantic tokens (`text-text-base`, `text-text-secondary`, `text-text-muted`)
- No hardcoded `border-gray-*` — replace with `border-border`
- All SVG icons: use `currentColor` fill/stroke so they inherit text color
- NZRegionMap SVG paths: use Tailwind fill tokens (not hardcoded hex fills)
