**1. Replace getRegionColor function** (currently ~lines 18-26):

Current (hardcoded blue hex):
```typescript
function getRegionColor(count: number, maxCount: number): string {
  if (count === 0 || maxCount === 0) return "#f3f4f6";
  const intensity = count / maxCount;
  if (intensity < 0.2) return "#dbeafe";
  if (intensity < 0.4) return "#93c5fd";
  if (intensity < 0.6) return "#3b82f6";
  if (intensity < 0.8) return "#2563eb";
  return "#1d4ed8";
}
```

Replace with teal CSS variable version:
```typescript
function getRegionFill(count: number, maxCount: number): string {
  if (count === 0 || maxCount === 0) return "var(--color-surface-raised)";
  const intensity = count / maxCount;
  if (intensity < 0.25) return "color-mix(in srgb, var(--color-accent) 20%, transparent)";
  if (intensity < 0.5) return "color-mix(in srgb, var(--color-accent) 50%, transparent)";
  if (intensity < 0.75) return "color-mix(in srgb, var(--color-accent) 80%, transparent)";
  return "var(--color-accent)";
}
```

Update all call sites from `getRegionColor(...)` to `getRegionFill(...)`.

**2. SVG path stroke** — Replace hardcoded stroke colors:
- Default stroke: `stroke="var(--color-border)"` with `strokeWidth={1.5}`
- Hovered stroke: `stroke="var(--color-accent)"` with `strokeWidth={2}`  (using accent-light if available as CSS var)
- Selected stroke: `stroke="var(--color-accent)"` with `strokeWidth={3}`

**IMPORTANT (Pitfall 4):** SVG paths use inline `fill` prop — do NOT use Tailwind `fill-*` classes on the same element. Use CSS custom property strings in the `fill` prop directly.

**3. Tooltip restyling** — Replace `bg-gray-900 text-white` with `bg-primary text-text-inverse` (creates natural contrast in both modes).
