## Spacing Scale

Standard 4px base unit inherited from Phase 1. No exceptions for this phase.

| Token | Value | Usage in Admin Pages |
|-------|-------|---------------------|
| xs | 4px | Icon gaps in nav items, tight inline padding in badges |
| sm | 8px | Gap between stat card label and value; table cell vertical padding |
| md | 16px | Card internal padding (compact cards: `p-4`); form field padding |
| lg | 24px | Card internal padding (standard cards: `p-6`); section spacing in billing settings |
| xl | 32px | Between stat grid and table sections (`space-y-8`) |
| 2xl | 48px | Not used at this scale in admin — admin is data-dense |
| 3xl | 64px | Not used in admin pages |

Exceptions: Touch targets minimum 44px (`min-h-[44px]`) for all table row action buttons. Sidebar nav items use `py-2 px-3` (8px + 12px) — 12px is between sm and md but acceptable for nav items with icon+text.

**Source:** DESIGN.md Spacing section. CONTEXT.md D-05 (admin content areas use `max-w-content` where appropriate).

## Typography

Admin pages are data-dense. The active type sizes per screen are 3–4 stops from the full scale.

| Role | Size | Font Family | Weight | Line Height | Usage |
|------|------|-------------|--------|-------------|-------|
| Page heading | 24px (xl) | General Sans (`font-display`) | 600 | 1.3 | Page title ("Dashboard", "Users", "Billing Dashboard") |
| Section heading | 18px (lg) | DM Sans (`font-sans`) | 600 | 1.5 | Table section headers ("Subscriptions", "Escrow Deposits") |
| Body / table cells | 14px (sm) | DM Sans (`font-sans`) | 400 | 1.5 | Table data, card descriptions, user emails |
| Label / badge / table header | 12px (xs) | DM Sans (`font-sans`) | 600 | 1.4 | Column headers (`uppercase tracking-wide`), badge text, form labels |

**Stat values (numeric):** 24px (xl) General Sans 600 with `tabular-nums` — same pattern as dashboard stat cards in Phase 6.

**Monetary values:** 14px DM Sans 400 `font-mono tabular-nums` — JetBrains Mono renders dollar amounts in table columns. Right-aligned.

**Weights used (exactly 2 per family):**
- General Sans: 600 (page headings, stat values), 700 not used in admin (no hero-scale headings)
- DM Sans: 400 (body, table cells, metadata), 600 (section headings, labels, nav items)
- JetBrains Mono: 400 (monetary values in tables)

**Source:** DESIGN.md Typography scale. CONTEXT.md D-07 (numbers `tabular-nums`), D-11 (table headers `text-muted uppercase tracking-wide text-xs`), D-12 (monetary values `tabular-nums font-mono`, right-aligned).

## Color

| Role | Value | CSS Token | Usage |
|------|-------|-----------|-------|
| Dominant (60%) | `#fafaf7` (light) / `#0f0f1a` (dark) | `bg-bg` | Admin page background (main content area) |
| Secondary (30%) | `#ffffff` (light) / `#1a1a2e` (dark) | `bg-surface` | Admin header bar (`bg-primary`), sidebar (`bg-surface`), stat cards (`.card`), table wrappers (`.card`) |
| Accent (10%) | `#0d9488` | `text-accent` / `bg-accent` | Active sidebar nav item (`bg-accent-light text-accent`), primary action buttons, badge-info backgrounds |
| Destructive | `#dc2626` | `bg-error` / `text-error` | Destructive buttons in user management; badge-error for PAST_DUE subscriptions and EXPIRED escrows |

**Admin header bar:** Uses `bg-primary` (#1a1a2e light / #fafaf7 dark — the "primary" token inverts in dark mode per Phase 1). "Admin Panel" title in `text-text-inverse`. This keeps the header visually distinct from user-facing pages.

**Accent reserved for:**
1. Active sidebar nav item background (`bg-accent-light`) and text (`text-accent`)
2. Primary action buttons (`btn-primary` — `bg-accent hover:bg-accent-hover`)
3. `badge-info` backgrounds (teal tint `bg-accent-light`)
4. Form field focus rings in billing settings and email template editor (`ring-accent-light`)
5. Accent left-stripe on emphasized stat cards (`border-l-4 border-accent` — Total Users card only per D-08)

Accent is NOT used for: table dividers, stat value text color (uses `text-primary`), section heading text, or general backgrounds.

**Status badge color mapping (source: CONTEXT.md D-14 through D-22):**

Escrow statuses:
- PENDING → `badge-warning` (amber `bg-secondary-light text-warning`)
- HELD → `badge-info` (teal `bg-accent-light text-accent`)
- RELEASED → `badge-success` (green `bg-success-light text-success`)
- REFUNDED → `badge-neutral` (gray `bg-surface-raised text-text-secondary`)
- EXPIRED → `badge-error` (red `bg-error-light text-error`)

Subscription statuses:
- ACTIVE → `badge-success`
- CANCELED → `badge-neutral`
- PAST_DUE → `badge-error`
- TRIALING → `badge-info`

User role badges:
- ADMIN → `badge-warning` (replaces current `bg-purple-100 text-purple-700`)
- USER (default) → `badge-neutral`

**Source:** CONTEXT.md D-02 (header colors), D-03 (sidebar colors), D-06 (uniform stat cards), D-07 (stat value text-primary), D-08 (accent left-stripe on one card), D-14–D-22 (badge mappings), D-28 (hardcoded role badge replacement).

## Financial Data Table Pattern

All billing tables (subscriptions, escrows) follow this pattern:

```
.card (table wrapper — no extra padding, overflow-hidden)
  <table class="w-full">
    <thead>
      <tr class="border-b border-border">
        <th class="px-6 py-3 text-left text-xs font-semibold text-text-muted uppercase tracking-wide">
          Column Name
        </th>
      </tr>
    </thead>
    <tbody class="divide-y divide-border">
      <tr class="hover:bg-surface-raised transition-colors">
        <td class="px-6 py-4 text-sm text-text-base">
          value
        </td>
      </tr>
    </tbody>
  </table>
```

**Monetary values in table cells:**
```
<td class="px-6 py-4 text-right font-mono text-sm text-text-base tabular-nums">
  $299.00
</td>
```

**Date values in table cells:**
```
<td class="px-6 py-4 text-sm text-text-secondary tabular-nums">
  2026-03-30
</td>
```

**Count/numeric values in table cells (non-monetary):**
```
<td class="px-6 py-4 text-sm text-text-base tabular-nums">
  42
</td>
```

**Source:** CONTEXT.md D-09 (no stripes), D-10 (hover:bg-surface-raised), D-11 (header style), D-12 (monetary: font-mono tabular-nums right-aligned), D-13 (dates: text-secondary tabular-nums).

## Responsive Behavior

Admin pages are desktop-primary (data tables, sidebars) but must not break at smaller breakpoints.

| Breakpoint | Layout behavior |
|------------|----------------|
| xl (1280px+) | Full sidebar + content layout |
| lg (1024px) | Full sidebar + content layout, stat grids may compress |
| md (768px) | Sidebar collapses consideration: keep sidebar at w-64 but allow table horizontal scroll |
| sm (640px) | Tables scroll horizontally (`overflow-x-auto` on table wrapper) — do not collapse to cards |

Table wrapper pattern for responsive scroll:
```
<div class="card overflow-x-auto">
  <table class="w-full min-w-[640px]">
    ...
  </table>
</div>
```
