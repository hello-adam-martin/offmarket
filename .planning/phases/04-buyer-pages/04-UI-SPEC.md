## Spacing Scale

Source: DESIGN.md + tailwind.config.ts (4px base unit). No new tokens added this phase.

| Token | Value | Usage in Phase 4 |
|-------|-------|-----------------|
| xs | 4px | Icon gaps (`gap-1`), badge inner padding, badge gap overrides (minimum grid-aligned value) |
| sm | 8px | Compact element spacing, button padding (`px-3 py-1.5` for btn-sm) |
| md | 16px | Default element spacing, card inner gap, form field gap |
| lg | 24px | Section padding within cards, `p-6` on `.card` |
| xl | 32px | Layout vertical rhythm between card sections |
| 2xl | 48px | Major section breaks on create page |
| 3xl | 64px | Page-level top/bottom padding (`py-8` = 32px min, page-level uses py-8 to py-12) |

Exceptions:
- Touch targets for toggle buttons (property type, feature selection): minimum 36px height enforced via badge-info padding (`py-0.5` = 2px) — acceptable for desktop-primary buyer workflow. No mobile tap target override needed.
- Max content width: `max-w-content` (1120px) on all three page containers. No `max-w-3xl`, `max-w-xl`, `max-w-4xl` permitted — these are the exact classes being replaced per D-12.

## Typography

Source: DESIGN.md typography scale, confirmed in tailwind.config.ts fontSize entries.

| Role | Size | Weight | Line Height | Font | Tailwind Class | Usage in Phase 4 |
|------|------|--------|-------------|------|----------------|-----------------|
| Body | 16px | 400 | 1.6 | DM Sans | `text-md` | Form descriptions, card body text, metadata, hints, postcard cost, allowance usage line, secondary/supporting text |
| Label | 12px | 600 | 1.4 | DM Sans | `text-xs` (or `.label`) | Form labels (via `.label` class), badge text |
| Heading | 24px | 600 | 1.3 | General Sans | `text-xl font-display` | Card section headings on create form, page headings |
| Page Title / Stat | 32px | 600 | 1.2 | General Sans / DM Sans | `text-2xl font-semibold font-display` / `text-2xl font-semibold tabular-nums text-accent` | Page titles ("Create Wanted Ad", "My Ads", "Postcards") and match count large number on My Ads cards (D-06) |

Rules:
- General Sans (`font-display`) used only for page titles and section headings (h1–h3 equivalents).
- DM Sans (`font-sans`, default) used for all body, labels, metadata, and data displays.
- Secondary/supporting text (metadata, hints, postcard cost, allowance usage line) uses `text-md text-secondary` — same 16px size as body, with `text-secondary` color providing visual hierarchy instead of a separate size.
- `tabular-nums` class applied directly on any element containing numeric output (budget, match count, postcard count, usage fractions). Do not wrap in a separate `<span>` unless the element itself contains mixed text.
- `.label` CSS class encapsulates: `text-xs font-semibold text-text-secondary mb-1 uppercase tracking-wide` — use for all `<label>` elements in the create form.
- Data/numbers use `text-md font-normal tabular-nums` (weight 400, size 16px).

## Color

Source: DESIGN.md Color section, CSS variables in globals.css, tailwind.config.ts.

### 60/30/10 Split

| Role | Value (light) | Value (dark) | Usage |
|------|--------------|--------------|-------|
| Dominant (60%) | `#fafaf7` (`bg-bg`) | `#0f0f1a` (`bg-bg`) | Page background (`bg-bg`), all three buyer pages |
| Secondary (30%) | `#ffffff` (`bg-surface`) / `#f5f5f0` (`bg-surface-raised`) | `#1a1a2e` / `#252540` | Cards (`.card`), info sections, usage summary bar, skeletons |
| Accent (10%) | `#0d9488` (`accent`) | `#0b8276` (reduced saturation) | Reserved list below |
| Destructive | `#dc2626` (`error`) | `#dc2626` | Delete button text and error states only |

### Accent Reserved For (explicit list)

The teal accent (`text-accent`, `bg-accent`, `border-accent`, `text-accent-hover`) is permitted only on:

1. Primary CTA buttons (`.btn-primary` — bg-accent background)
2. Match count large number on My Ads cards (`text-2xl font-semibold text-accent tabular-nums`)
3. Active selection card border (`border-accent`) and tint (`bg-accent-light`) on target type cards
4. Active property type and feature toggle badge (`badge-info` — bg-accent-light text-accent)
5. Location tag badges (`badge-info`)
6. Postcard allowance "N free left" badge (`badge-info`)
7. Active/selected state icon color (`text-accent`)
8. Links within the create form (upgrade prompt: `text-accent hover:text-accent-hover`)
9. Budget display text on My Ads cards (`text-accent font-medium tabular-nums`)
10. Remove button (`×`) inside location tags (`text-accent hover:text-accent-hover`)

Accent is NOT used on: decorative backgrounds, gradient fills, icon containers (use `bg-accent-light` for tinted containers, `bg-surface-raised` for neutral containers), body text, metadata, or any element not in the list above.

## Interaction Contracts

### Target Type Selection Cards (create/page.tsx, lines 356–435)

Two-card toggle: "General Area" vs "Specific Property".

| State | Border | Background | Ring | Icon bg | Icon color | Title color |
|-------|--------|-----------|------|---------|-----------|-------------|
| Selected | `border-accent border-2` | `bg-accent-light` | `ring-2 ring-accent-light` | `bg-accent-light` | `text-accent` | `text-accent` |
| Unselected | `border-border border-2` | `bg-surface` | none | `bg-surface-raised` | `text-muted` | `text-primary` |
| Disabled (at limit) | `border-border border-2` | `bg-surface-raised opacity-75` | none | `bg-surface-raised` | `text-muted` | `text-muted` |

### Property Type / Feature Selection Badges (create/page.tsx, lines 633–726)

| Inactive | `badge-neutral cursor-pointer hover:bg-surface-raised transition-colors` | bg-surface-raised, text-secondary, rounded-sm |

Remove all `px-*`, `py-*`, `rounded-*`, `text-*`, `font-*` classes when switching to `.badge-*` — the class provides all sizing.

## Anti-Patterns (Blocked in Phase 4)

These patterns are explicitly banned. Flag and replace on sight:

| Pattern | Replacement |
|---------|-------------|
| `bg-gradient-to-r` anywhere | `bg-surface-raised` (flat) |
| `rounded-full` on badges or selection toggles | `rounded-sm` (4px) via `.badge-*` class |
| `primary-{number}` color classes (e.g. `primary-50`, `primary-600`) | `accent`, `accent-light`, `accent-hover` per intent |
| `gray-*` hardcoded colors | `text-primary`, `text-secondary`, `text-muted`, `bg-surface`, `bg-surface-raised`, `border-border` |
| `red-*` hardcoded colors | `text-error`, `bg-error-light`, `border-error` |
| `green-*` hardcoded colors | `text-success`, `bg-success-light` |
| `amber-*` / `yellow-*` hardcoded colors | `badge-warning` or `text-warning`, `bg-secondary-light` |
| `blue-*` hardcoded colors | `badge-info` |
| `max-w-3xl`, `max-w-xl`, `max-w-4xl` | `max-w-content` |
| `btn-outline` class (undefined) | `btn-secondary` |
| Extra `px-*/py-*` alongside `.badge-*` classes | Remove — badge-* provides its own padding |
| `bg-accent/10` or `bg-error/10` opacity modifier syntax | `bg-accent-light`, `bg-error-light` (CSS variable opacity fails in @apply, Pitfall 4) |
| `statusInfo.color` field reference | `statusInfo.badgeClass` after STATUS_LABELS refactor |
| `font-bold` on page titles or stat numbers | `font-semibold` — only 2 weights (400 and 600) are permitted |
| `text-sm` for secondary/metadata text | `text-md text-secondary` — 16px body size with color hierarchy, not separate size |
| `gap-0.5` (2px) on badge containers | `gap-1` (4px) — minimum grid-aligned gap value |

## Dark Mode Contract

All token classes in this spec are dark-mode aware via the `.dark` selector in globals.css. No additional dark mode work is required beyond using semantic tokens.

Key dark mode values for buyer pages:
- `bg-surface` → `#1a1a2e` in dark
- `bg-surface-raised` → `#252540` in dark
- `bg-accent-light` → `#134e4a` in dark (teal tint stays legible)
- `border-border` → `#374151` in dark
- `text-primary` (via `--color-text`) → `#fafaf7` in dark

Do not use `dark:` variant classes. Use semantic tokens only — they switch automatically.
