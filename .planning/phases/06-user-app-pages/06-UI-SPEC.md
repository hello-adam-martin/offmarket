## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| 2xs | 2px | Dividers, hairline gaps |
| xs | 4px | Icon gaps, inline badge padding, tight element spacing |
| sm | 8px | Compact element spacing, button inner padding (sm variant) |
| md | 16px | Default element spacing, card inner padding, form field gaps |
| lg | 24px | Section padding, card gap in grids |
| xl | 32px | Layout gaps between major sections |
| 2xl | 48px | Major section breaks (page-level) |
| 3xl | 64px | Page-level top/bottom spacing |

Exceptions:
- Touch targets on icon-only actions: minimum 44px (WCAG AA)
- Message bubble padding: 12px horizontal, 8px vertical (between sm and md — 4px base compliant)
- Progress bar track height: 8px (sm)

Source: DESIGN.md Spacing section; tailwind.config.ts spacing extension.

## Typography

| Role | Font | Size | Weight | Line Height | Class |
|------|------|------|--------|-------------|-------|
| Body text | DM Sans | 16px (md) | 400 | 1.6 | `text-md font-sans` |
| Secondary / metadata | DM Sans | 14px (sm) | 400 | 1.5 | `text-sm font-sans` |
| Labels / badges | DM Sans | 12px (xs) | 600 | 1.4 | `text-xs font-semibold uppercase tracking-wide` |
| Subsection headings | General Sans | 24px (xl) | 600 | 1.3 | `text-xl font-display` |
| Section headings | General Sans | 32px (2xl) | 700 | 1.2 | `text-2xl font-display` |
| Data / numbers | DM Sans | 16–24px (md–xl) | 700 | 1.3 | `tabular-nums font-bold` |
| Mono / claim codes | JetBrains Mono | 14–16px | 400–500 | 1.5 | `font-mono tabular-nums` |

Rules:
- All h1–h3 use `font-display` (General Sans). All body, labels, data use `font-sans` (DM Sans).
- All numeric displays (prices, counts, percentages, claim codes) must have `tabular-nums` on the containing element.
- Timestamps use `text-sm text-muted`.
- Two weights only in body: 400 (regular) and 600 (semibold/labels). Bold (700) reserved for stat values and headings.

Source: DESIGN.md Typography section; tailwind.config.ts fontSize config.

## Color

| Role | Light mode value | Dark mode value | Usage |
|------|-----------------|-----------------|-------|
| Dominant (60%) | `#fafaf7` (`bg-background`) | `#0f0f1a` | Page background |
| Secondary (30%) | `#ffffff` / `#f5f5f0` (`bg-surface` / `bg-surface-raised`) | `#1a1a2e` / `#252540` | Cards, sidebar, nav, nested surfaces |
| Accent (10%) | `#0d9488` (`accent`) | `#0b8276` | Reserved elements — see below |

**Accent reserved for:**
- Primary CTA buttons (`btn-primary`: bg-accent)
- Active billing toggle tab (text-accent)
- Pro pricing card border highlight (border-accent)
- Feature checklist checkmarks on available features (text-accent)
- Upgrade CTA card left border stripe (border-l-4 border-accent)
- Quick action card hover state border (hover:border-accent)
- Message bubbles for own messages (bg-accent text-white)

- Text links and interactive text (text-accent)

**Accent NOT used for:**
- Decorative backgrounds
- Section dividers
- Icon containers
- Status indicators (use badge-success/warning/error/neutral)

## Page-Level Design Contracts

### Dashboard (USER-01)

**Layout:** `max-w-content` centered container. Stat cards in responsive grid (sm: 1col, md: 2col, lg: 4col). Quick actions below as horizontal card list.

**Stat cards:** `.card` class. Numbers in `tabular-nums font-bold text-xl font-display`. Label in `.label`. No icon containers.

**Upgrade CTA card (free users):** `.card` + `border-l-4 border-accent`. No gradient background. Children: "Free Plan" `badge-neutral`, usage count `tabular-nums`, progress bar, "Upgrade" `btn-primary`. Remove: `bg-gradient-to-r from-purple-50 via-primary-50 to-accent-50`.

**Quick action links:** `bg-surface border border-border rounded-md p-4` with `hover:border-accent transition-colors`. Text `text-primary` / `text-secondary`. Arrow icon `text-muted`.

**Pro badge:** `badge-info` (replaces `bg-primary-100 text-primary-700 rounded-full`).

**Usage progress bar:** Track `bg-surface-raised h-2 rounded-full`. Fill `bg-accent` / `bg-warning` / `bg-error` per threshold.

Source: CONTEXT.md D-11, D-12, D-13, D-14.

**Save button:** `btn-primary` at bottom of each section. Destructive account actions use `btn-destructive`.

**Success/error feedback:** Inline below form using semantic color text (`text-success` / `text-error`), not toasts.

**Action buttons:**

- Mark Complete: `btn-secondary`

**Reply form:** `.input` for textarea. `btn-primary` for submit.

**Status badge:** `badge-*` per STATUS_LABELS mapping.

## Dark Mode Requirements

All Phase 6 pages must render correctly with the `.dark` class active. The following CSS variable overrides are already defined in `globals.css .dark` block and will apply automatically when semantic tokens are used:

- Background: `#0f0f1a` (bg-background)
- Surface: `#1a1a2e` (bg-surface)
- Surface Raised: `#252540` (bg-surface-raised)
- Border: `#374151` (border-border)
- Accent: `#0b8276` (reduced saturation per DESIGN.md)
- success-light, error-light, secondary-light: dark-mode overrides defined

Dark mode compliance is automatic when hardcoded `gray-*` and `primary-*` classes are replaced with semantic tokens. No per-page dark: variant overrides should be needed.

Source: globals.css .dark block; DESIGN.md Dark Mode section.

## Accessibility Requirements

- All interactive elements: minimum 44px touch target (WCAG AA)
- Focus rings: `focus:ring-2 focus:ring-accent-light focus:ring-offset-2` (already in `.btn` and `.input` base classes)
- Color contrast: semantic tokens satisfy WCAG AA — do not override with hardcoded colors
- Avatar fallback initials: include `aria-label` with user name
- Status badges: include `aria-label` describing status for screen readers
- Message thread: use `role="log"` on message container, `aria-live="polite"` for new messages
- Destructive confirmation dialogs: use `role="alertdialog"` with `aria-describedby`
