## Spacing Scale

Declared values (base unit: 4px — source: DESIGN.md):

| Token | Value | Tailwind Class | Usage |
|-------|-------|----------------|-------|
| 2xs | 2px | — | Fine details only (thin separators, icon micro-gaps) — not for layout |
| xs | 4px | p-1 / gap-1 | Icon gaps, badge internal padding |
| sm | 8px | p-2 / gap-2 | Compact element internal spacing, stat row gaps |
| md | 16px | p-4 / gap-4 | Default element spacing, form field internal padding |
| lg | 24px | p-6 / gap-6 | Card internal padding, section padding |
| xl | 32px | p-8 / gap-8 | Layout gaps, between-card spacing |
| 2xl | 48px | p-12 / gap-12 | Major section breaks within a page |
| 3xl | 64px | p-16 / gap-16 | Page-level vertical section spacing |

Exceptions:
- Touch targets: minimum 44px on all interactive elements (WCAG 2.5.5) — enforce via `min-h-[44px]` on buttons and links.
- Property image thumbnails: 80px × 60px (20:15 ratio) — specific to property listing cards.

**Source:** DESIGN.md Spacing section, Phase 1 UI-SPEC.

## Typography

| Role | Font | Size | Weight | Line Height | Token / Class |
|------|------|------|--------|-------------|---------------|
| Display / Hero | General Sans | 48px | 700 | 1.1 | text-3xl (Tailwind custom) |
| Section heading | General Sans | 32px | 700 | 1.2 | text-2xl |
| Subsection / stat value | General Sans | 24px | 600 | 1.3 | text-xl |
| Lead / body large | DM Sans | 18px | 400 | 1.5 | text-lg |
| Body | DM Sans | 16px | 400 | 1.6 | text-md / text-base |
| Secondary / metadata | DM Sans | 14px | 400 | 1.5 | text-sm |
| Labels / badges | DM Sans | 12px | 600 | 1.4 | text-xs uppercase tracking-wide |

**Active weights in this phase:** 400 (regular), 600 (semibold), 700 (bold).
**Data displays:** All numeric values (demand counts, match scores, budget ranges, RV, estimated value) use `tabular-nums` class or `font-variant-numeric: tabular-nums` — DM Sans weight 700, `text-accent` for key figures.

**Phase-specific applications:**
- Owner landing page hero: `text-3xl` General Sans 700 — left-aligned (not centered).
- Demand summary total: `text-xl` (24px) General Sans 600 `text-accent tabular-nums font-bold`.
- Match score percentage: `text-sm` DM Sans 700 `text-accent tabular-nums`.
- Property card demand count: `text-sm` DM Sans 700 `text-accent tabular-nums`.
- Section card headings on register form: `text-base` DM Sans 600 `text-primary`.

**Source:** DESIGN.md Typography section, confirmed in globals.css and tailwind.config.ts.

## Color

| Role | Light Mode Value | Dark Mode Value | CSS Variable | Usage |
|------|-----------------|-----------------|--------------|-------|
| Dominant (60%) — background | #fafaf7 | #0f0f1a | --color-bg | Page background, all pages |
| Secondary (30%) — surfaces | #ffffff / #f5f5f0 | #1a1a2e / #252540 | --color-surface / --color-surface-raised | Cards (.card), form section cards, modal panels, match item rows |
| Accent (10%) — teal | #0d9488 (hover: #0f766e) | #0b8276 (hover: #0d6b62) | --color-accent / --color-accent-hover | See accent reserved list below |
| Border | #e5e5e0 | #374151 | --color-border | All card borders, input borders, dividers |
| Text primary | #1a1a2e | #fafaf7 | --color-text / text-primary | All headings, body copy, labels |
| Text secondary | #6b7280 | #9ca3af | --color-text-secondary | Metadata, subtext, helper text, "direct · criteria" stats row |
| Text muted | #9ca3af | #6b7280 | --color-text-muted | Placeholder text, disabled labels |
| Destructive | #dc2626 (error-light bg: #fee2e2) | #dc2626 (error-light bg: #7f1d1d) | --color-error / --color-error-light | Delete property button, error messages |
| Success | #059669 (success-light bg: #d1fae5) | #059669 (success-light bg: #064e3b) | --color-success / --color-success-light | Success badge on property status (if applicable) |
| Warning / amber | #d97706 (secondary-light bg: #fef3c7) | #d97706 (secondary-light bg: #451a03) | --color-secondary / --color-secondary-light | Warning badges only |

**Accent (#0d9488) is reserved exclusively for:**
- Primary CTA buttons (`btn-primary`) — "Register Your Property", "View Details", "Contact Buyer"
- Demand total count value (text-accent + tabular-nums)
- Match score percentage values (text-accent + tabular-nums)
- Property card demand count link text (text-accent)
- `badge-info` badge text (accent on accent-light background) — feature toggles, match-criteria tags
- Focus rings on all form inputs (focus:border-accent focus:ring-1 focus:ring-accent-light)
- Hover border on property cards (hover:border-accent)
- Demand summary panel left border stripe (border-l-4 border-accent)
- Direct interest card left border stripe (border-l-4 border-accent)
- "View Details" links on my-properties listing (text-accent)

**Accent is NOT used for:**
- Informational body copy
- Section headings
- Background fills on large surfaces
- Loading skeletons (use bg-surface-raised)

**Source:** DESIGN.md Color section, CONTEXT.md decisions D-01 through D-30.

## Page-Level Interaction Contracts

### /owner — Landing Page

**Layout:** `max-w-content` (1120px) container. Server component (no auth required).

**Hero section:**
- Left-aligned text (NOT centered) — AI slop violation if centered.
- Heading: `text-3xl` General Sans 700 `text-primary`.
- Subtext: `text-lg` DM Sans 400 `text-secondary`.
- CTA button: `btn-primary btn-lg` — label: "Register Your Property".
- No gradient background. No SVG pattern. bg-bg page background.

**Benefits section:**
- Remove icon-in-colored-circle pattern (AI slop blacklist).
- Use left-aligned icon + text rows: `flex items-start gap-4`, plain Lucide icon (24px, text-accent), heading in `text-base font-semibold text-primary`, body in `text-sm text-secondary`.
- No centered card grid with identical structure.

**DemandChecker component:** Already restyled (Phase 3). Use as-is.

**CTA section:**
- Flat `bg-surface-raised` card with `border-border`. No gradient.
- Replace `bg-gradient-to-br from-accent-50 to-primary-50`.
- CTA button: `btn-primary`.

**FAQ section:**
- Cards use `.card` class (already present).
- Swap `text-gray-900` → `text-primary`, `text-gray-600` → `text-secondary`.

---

### /owner/register — Register Property Form

**Layout:** `max-w-content` (1120px) container. Multi-card section layout retained.

**Section cards:** Each section (Address, Details, Size, Features, Valuation) uses `.card` class.

**Section headings:** `text-base` / `text-lg` DM Sans 600 `text-primary`. Swap `text-gray-900`.

**Form inputs:** All use `.input` class. Labels use `.label` class.

**AddressAutocomplete:** Already restyled. Use as-is.

**Feature/property type selection badges:**
- Active state: `badge-info` — accent-light background, teal text, `rounded-sm` (4px).
- Inactive state: `badge-neutral` — surface-raised background, secondary text.
- Remove `rounded-full` pills. Remove `bg-primary-600` active fill.

**Image upload section:** `PropertyImageUpload` component already restyled. Use as-is.

**Submit button:** `btn-primary btn-lg` full-width or right-aligned — label: "Register Property".

**Loading state:** Spinner inline with button text, button disabled. No layout shift.

---

### /owner/my-properties — Property Listing

**Layout:** `max-w-content` (1120px) container.

**Property cards:**
- Use `.card` class with `hover:border-accent transition-colors`.
- Replaces `hover:border-primary-300`.

**Property type badge:** `badge-neutral` — surface-raised bg, secondary text, rounded-sm.
- Replaces inline `bg-gray-100 text-gray-700 rounded-full`.

**Demand count:**
- `text-accent font-bold tabular-nums` — prominent, large.
- Replaces conditional `text-primary-600` / `text-gray-400`.

**"View Details" link:** `text-accent hover:text-accent-hover` with underline on hover.
- Replaces `text-primary-600`.

**Delete button:** `text-error hover:text-error-hover text-sm` — destructive action.
- Replaces `text-red-600 hover:text-red-800`.
- Confirmation: inline confirm state (button changes to "Are you sure? Yes / Cancel" — no modal needed for this destructive action).

**Property image thumbnail:** 80×60px, `rounded-md` (8px), `object-cover`. bg-surface-raised when no image.

**Loading state:** Skeleton uses `bg-surface-raised animate-pulse rounded-md`. Replaces `bg-gray-200`.

**Empty state:** Reuse EmptyState component pattern (COMP-09). Heading: "No properties yet". Body: "Register your first property to start seeing buyer demand." CTA: `btn-primary` → "Register Your Property".

---

### /owner/properties/[id] — Property Detail

**Layout:** `max-w-content` (1120px) container.

**Back link:** `text-sm text-secondary hover:text-primary` with `← ` chevron or arrow icon (16px, inline).

**Demand Summary Panel:**
- `.card` with `border-l-4 border-accent` left stripe. No gradient.
- Replaces `bg-gradient-to-br from-primary-50 to-white border-primary-200`.
- Total buyer count: `text-xl` (24px) General Sans 600 `text-accent font-bold tabular-nums`.
- Direct vs criteria split: single inline row below total — "12 direct · 8 criteria" in `text-sm text-secondary tabular-nums`.
- No separate colored sub-cards (remove `bg-green-50` / `bg-blue-50` boxes).
- Budget range: label in `text-sm text-secondary`, value in `text-sm text-primary tabular-nums`.

**Match Score Display:**
- All match score percentages: `text-accent font-bold tabular-nums`. Single color — no green/yellow/orange tier coloring.
- Replaces current threshold-based color system.
- Match criteria tags (Location, Budget, Bedrooms, Features): `badge-info` — accent-light bg, accent text, rounded-sm.
- Replaces `bg-blue-100 text-blue-700`.

**Direct Interest Section:**
- `.card` with `border-l-4 border-accent`. No gradient.
- Replaces `border-2 border-green-200 bg-gradient-to-br from-green-50 to-white`.
- No icon-in-circle header decoration.
- Section heading: `text-base font-semibold text-primary`. Subtext: `text-sm text-secondary`.
- Contact button: `btn-primary` — label: "Contact Buyer".
- Replaces `bg-green-600 hover:bg-green-700`.

**Individual match item rows:**
- `bg-surface border border-border rounded-md p-4`.
- Replaces `bg-white border border-green-200`.

**PropertyImageUpload component:** Already restyled (Phase 2). Use as-is.

**ContactBuyerModal:** Already restyled (Phase 2). Use as-is.

**Loading state:** Page skeleton with `bg-surface-raised animate-pulse`. Replaces `bg-gray-200`.
